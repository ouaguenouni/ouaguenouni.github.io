# Presentations — the workflow (goodbye Beamer)

Talks live in `presentations/`. Each talk is a **folder** with a `slides.md`:

```
presentations/
  presentation.css       ← shared style tweaks (all talks)
  index.html             ← the list of talks (hand-maintained)
  _template/             ← copy this folder to start a new talk
    slides.md
  example-talk/
    slides.md            ← YOUR SLIDES, in Markdown    ← you edit this
    diagram.svg          ← any images/assets the talk uses
    index.html           ← GENERATED by generate.py    ← never edit by hand
```

You write **only Markdown** in `slides.md`. `generate.py` compiles it into the
reveal.js deck (`index.html`) — using the **same Markdown + math pipeline as
your articles**, so `$…$` / `$$…$$` LaTeX renders exactly like in your papers.
(reveal's own Markdown reader mangles backslashes; that's why we compile.)

---

## 1. Run it locally

```bash
./dev.sh
```

This serves the site **and** watches your files with **live reload**: edit a
`slides.md`, save, and the browser refreshes itself automatically. Open:

- Site:          <http://localhost:8000/>
- Presentations: <http://localhost:8000/presentations/>
- The example:   <http://localhost:8000/presentations/example-talk/>

To stop, press `Ctrl+C`.

> One-off build without the watcher: `./venv/bin/python generate.py`
> (rebuilds every article and every talk once).

---

## 2. The edit loop

1. Edit `presentations/example-talk/slides.md`
2. **Save**

That's it — **live reload** is on: saving recompiles the talk and the browser
refreshes itself automatically (no `Cmd+R`). Keep the editor and browser side by
side and just type + save.

## 3. Slide syntax (cheat sheet)

It's the same Markdown as your articles, plus a few slide separators:

| You want… | Write… |
|-----------|--------|
| A new slide | a line with only `---` |
| A sub-slide (press `↓` to reach) | a line with only `--` |
| Speaker notes | a line that is just `Note:` — text after it is the note |
| Reveal a bullet on click | `- point {: .fragment}` |
| Slide title metadata | a `title:` line in `---` front-matter at the very top |
| Slide background colour | `<!-- .slide: data-background="#0b3d2e" -->` at the top of the slide |
| Inline math | `$ \alpha + \beta $` |
| Display / multi-line math | `$$ … $$` (its own paragraph; `aligned`, `\\`, `_`, `^` all work) |
| Code (auto-highlighted) | a fenced block ` ```python … ``` ` |
| Two columns | see below |
| Image | `![alt](diagram.svg)` (put the file in the talk folder) |
| Quote | `> quoted text` |

**Two columns** need `markdown="1"` so the inner Markdown is compiled:

```html
<div class="columns" markdown="1">
<div markdown="1">

Left column **Markdown** here
- a bullet

</div>
<div markdown="1">

![right side](diagram.svg)

</div>
</div>
```

⚠️ Don't use `---` as a horizontal rule *inside* a slide — it starts a new
slide. Use `***` instead.

`example-talk/slides.md` demonstrates every one of these. Read it as the manual.

## 4. Figures, animation & media

The example deck has live slides for every item below — open it and look.

**Images / figures.** Drop the file in the talk folder; size with CSS:

```html
<figure>
<img src="diagram.svg" style="width:55%">
<figcaption>Figure 1 — a caption.</figcaption>
</figure>
```

Or just `![alt](diagram.svg)` for a quick image. PNG, JPG, SVG all work.

**Animated figures.** Animated **SVG**, **GIF**, and **video** play directly —
no plugin. Video can autoplay when its slide opens:

```html
<video data-autoplay src="clip.mp4"></video>
```

**Build steps (fragments).** Reveal items one click at a time with
`{: .fragment}` — your Beamer `\pause` (see §3).

**Auto-Animate (morphing).** Give matching elements the same `data-id` on two
slides marked `<!-- .slide: data-auto-animate -->`; reveal smoothly morphs
position, size, and colour between them. Perfect for building a diagram or
equation incrementally — the modern replacement for Beamer overlays.

**Slide transitions.** Set globally (already `slide`) or per slide:
`<!-- .slide: data-transition="fade" -->` (`none`, `fade`, `slide`, `convex`,
`concave`, `zoom`).

**Interactive figures.** The same `:::html` embed your articles use works in
slides — drop in a live Plotly / D3 / canvas widget that stays hoverable:

```
:::html plot.html :::
```

Put `plot.html` in the talk folder (load any library from a CDN inside it).
The deck fires a resize when a slide opens, so charts fit correctly even though
they're drawn while off-screen.

## 5. Presenting

Keyboard while a talk is open:

| Key | Action |
|-----|--------|
| `→` / `Space` | next step / slide |
| `←` | previous |
| `S` | **speaker view** — notes, timer, next-slide preview (opens a 2nd window) |
| `O` | overview grid of all slides |
| `F` | fullscreen (`Esc` to exit) |
| `B` / `.` | black out / pause |

For a real talk: go fullscreen (`F`), and on a second monitor open speaker view (`S`).

## 6. Export to PDF (the Beamer handout)

1. Open the talk with `?print-pdf` appended:
   `http://localhost:8000/presentations/example-talk/?print-pdf`
2. Open the **Print** dialog (`Cmd+P`) — use **Chrome** for best results.
3. Destination **Save as PDF**, margins **None**, **Background graphics ON**,
   paper **Landscape**.
4. Save → one page per slide, math and code intact.

---

## 7. Start a NEW talk

```bash
cp -r presentations/_template presentations/my-cool-talk
```

Then:

1. Edit `presentations/my-cool-talk/slides.md` (set the `title:`, write content).
2. Add it to the list in `presentations/index.html` (copy one `<article>` block).
3. With `./dev.sh` running, it builds automatically and is live at
   `http://localhost:8000/presentations/my-cool-talk/`.

## 8. Coming from Beamer

| Beamer | Here |
|--------|------|
| `\frame{ … }` | `---` between slides |
| `\pause`, `\onslide` | `{: .fragment}` on a bullet |
| `\section` divider | a slide with `data-background` |
| `align` / `equation` | the same LaTeX inside `$$ … $$` |
| `\note{ … }` | `Note:` lines (speaker view) |
| `columns` / `column` | `<div class="columns" markdown="1">` |
| recompile with `pdflatex` | save the `.md`; it rebuilds, you refresh |
| themes (Madrid, …) | swap the `theme/white.css` line in `presentation_template.html` |

Built on [reveal.js](https://revealjs.com) (loaded from a CDN — nothing to
install). The compiler is `generate.py`. reveal docs: <https://revealjs.com>.
